Email Automation


Email automation is the process of creating a system that sends emails automatically based on predefined schedules or triggers. To achieve this, we utilized Python and its powerful libraries. First, we set up the environment by installing necessary libraries like smtplib for sending emails, ssl for secure connections, and schedule for task scheduling. The email credentials were securely managed using environment variables to avoid exposing sensitive information. Next, we wrote a script to define the sender, recipient, email subject, and body. The EmailMessage class was used to structure the email, and the smtplib library, in combination with an SSL context, established a secure connection with the Gmail SMTP server to send the email. To automate the process, we integrated the schedule library, which allowed the email to be sent at regular intervals—every minute in our case. The script continuously runs in a loop, checking and executing the scheduled task. This approach demonstrates how email automation can be efficiently implemented for use cases like notifications, updates, or marketing campaigns.

The code automates email sending by using Python libraries like smtplib for SMTP communication, EmailMessage to structure the email, and ssl for secure connections. The sender, recipient, subject, and body are defined, and the send_email function connects to Gmail's SMTP server to send the email securely. The schedule library runs this function every minute in a continuous loop, ensuring emails are sent automatically at regular intervals.

Patient Health Monitoring System using IOT

The IoT-Based Patient Health Monitoring System is an innovative solution designed to enhance patient care through real-time monitoring of vital health parameters. This project leverages advanced Internet of Things (IoT) technologies to enable continuous tracking of patient health metrics, allowing healthcare providers to make informed decisions quickly and efficiently.

Objectives:

• To develop a system that continuously monitors vital health parameters, including heart rate, blood oxygen saturation (SpO2), body temperature, and environmental conditions (humidity and temperature).
• To provide real-time data transmission to healthcare professionals through a user-friendly mobile application.
• To enable early detection of potential health issues through timely alerts and notifications.
• To facilitate remote patient monitoring, reducing the need for frequent hospital visits and improving the overall efficiency of healthcare delivery.

System Components/Sensor
• MAX30100/102 Pulse Oximeter: Measures heart rate and SpO2 levels.
• DS18B20 Temperature Sensor Monitors body temperature.
• DHT11 Humidity & Temperature Sensor Tracks ambient humidity and temperature.

Microcontroller
• ESP32: Acts as the central processing unit, collecting data from sensors and transmitting it to the cloud via Wi-Fi.

Cloud Platform:
• Blynk: An IoT platform that enables real-time monitoring and data visualization through a mobile application.
